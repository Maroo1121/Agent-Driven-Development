# The ADD Manifesto

## Agent-Driven Development — 10 Principles for Disciplined Agent-Assisted Engineering

**Version 0.1.0**

---

We are engineers who build software with AI agents.

We have learned — through experience, failure, and iteration — that powerful agents without disciplined methodology produce inconsistent, unreliable, and unmaintainable results.

We believe that working with AI agents is a new engineering discipline that requires its own rigor, its own artifacts, and its own non-negotiable standards.

We propose these ten principles as the foundation of that discipline.

---

### 1. Context over Prompt

A well-structured context outperforms a clever prompt every time. The quality of agent output is bounded by the quality of the context it receives — not by the ingenuity of the instruction. Invest in context packaging before prompt engineering.

### 2. Constraints over Creativity

Unconstrained agents produce unpredictable outputs. Precision, boundaries, and explicit formatting rules consistently yield better results than open-ended freedom. Creativity within constraints is engineering; creativity without constraints is gambling.

### 3. Verification over Trust

Agent output is a hypothesis, not a deliverable. Every execution must pass through a verification gate with explicit, pre-defined acceptance criteria before it is considered valid. Trust is the output of repeated verification, never the input.

### 4. Explicit Intent over Implicit Assumption

If the agent has to infer your intent, you have already introduced risk. Every task must begin with a clear, written statement of intent that leaves no room for interpretation. Ambiguity is a bug in your process, not a feature of the agent.

### 5. Structured Iteration over Random Retry

Re-running the same prompt and hoping for a better result is not iteration — it is noise. Every retry must change exactly one variable: the context, the constraints, the scope, or the prompt. Structured iteration converges; random retry diverges.

### 6. Artifacts over Memory

Conversations disappear. Context windows reset. Agent memory is unreliable. The only durable knowledge is written knowledge. Every decision, every constraint, every validation result must be captured in a persistent artifact. If it isn't written down, it doesn't exist.

### 7. Human Gate over Full Autonomy

Agent autonomy is a privilege, not a right. It is earned through demonstrated reliability across verified executions. Until that threshold is met, every critical output passes through a human review gate. Autonomy scales with trust; trust scales with evidence.

### 8. Scope Control over Scope Creep

One task, one cycle. An agent that tries to solve everything at once solves nothing reliably. Tight scope produces verifiable output. Scope creep is the primary cause of agent failure — not model limitations.

### 9. Failure Analysis over Blame

When agent output fails validation, the failure belongs to the methodology, not the agent. Every failure is a signal: insufficient context, ambiguous constraints, missing validation criteria, or incorrect scope. Failure analysis improves the system; blame improves nothing.

### 10. Reproducibility over Brilliance

A brilliant result that cannot be reproduced is an anecdote. A mediocre result that can be systematically improved is engineering. ADD optimizes for reproducible processes over exceptional one-time outputs. The goal is a reliable system, not a lucky prompt.

---

## The Commitment

By adopting ADD, we commit to:

- **Never executing** without explicit validation criteria.
- **Never prompting** without structured context.
- **Never iterating** without formalized feedback.
- **Never shipping** without an execution log.
- **Never trusting** without a verification gate.
